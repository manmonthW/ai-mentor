# 珂以这样玩AI — 个人品牌网站规划方案 v1

> 目标:把现有 ai-mentor 站点,升级为一个**全面、完整、高质量**的个人 IP / 品牌主站。
> 品牌名:**珂以这样玩AI**(「王珂 · 可以 · 这样玩 AI」的三重双关,珂字为姓名字,非错别字)。
> 原则:突出个人真实作品,排除公司(Ericsson/IMS)特有应用。

---

## 一、品牌定位与策略

### 1. 一句话定位
> **一个用 AI 把想法当天变成产品的独立创造者 —— 珂。跟着珂,你也可以这样玩 AI。**

抛弃旧站生硬的「高级AI赋能专家」标签,换成有辨识度、可感知的人设:**高产、好玩、能落地的 AI 造物者**。你的护城河不是头衔,而是**几十个真实上线的应用 + 独特的方法(蒸馏思维/名著为 Skill)+ 法律与一人公司的纵深**。

### 2. 三大支柱(品牌的"实")
| 支柱 | 含义 | 代表作 |
|---|---|---|
| **造(Builder)** | 40+ 已部署应用,证明"玩 AI"的规模与速度 | Judgment Analyzer、LexSmile、智医通、VoltGrid、养老金系列… |
| **蒸馏(Distiller)** | 把「人的思维」「经典名著」提炼成可执行 Agent Skill —— 你最独特的方法论 | 女娲 Skill(蒸馏思维)、仓颉 Skill(蒸馏名著)、PPT/Slides Skill |
| **应用于法律 & 一人公司(Vertical)** | 用法律人背景切入法律AI + 一人公司(OPC)AI 运营,做别人做不了的纵深 | SuitAgent、专利代理师备考、小达报税、LexSmile |

外加一层能力:**教(Educator)** —— 通用/教培/物流行业 AI 培训,把上面三样变现。

### 3. 目标受众(网站要同时服务四类人)
1. **想学会用 AI 造东西的人**(核心粉丝)——"你也可以"的对象,靠作品和方法吸引。
2. **法律从业者 / 一人公司创始人**(垂直高价值)——你的服务/定制/培训买家。
3. **企业与机构培训采购**(B2B 商单)——行业 AI 培训的线索来源。
4. **同行 / 招聘方 / 合作方**(背书与机会)——靠专业深度建立信任。

### 4. 品牌语气(Voice)
好玩、真诚、动手派、去术语化。少讲"赋能"这类空词,多用"我造了个 X,它能……你也能"。中文为主,允许适度英文技术词。

## 二、内容资产盘点(取 / 舍)

### ✅ 主打(皇冠上的宝石,重点包装)
- **Skill 蒸馏**:女娲、仓颉(母 Skill 叙事)、PPT/Frontend Slides、微信公众号 Skill、Youtube Clipper
- **法律 AI**:Judgment Analyzer(裁判文书分析)、SuitAgent(10-Agent 诉讼系统)、专利代理师备考、LexSmile
- **一人公司(OPC)**:小达报税(已上线)、智能合同/合规/业务(规划)
- **思维工具**:Insight Vector / 维度罗盘、高维认知伴侣
- **精选自研/Fork**:Banana Slides、EchoTrace、WeKnora(RAG)、智医通、养老金系列
- **"数量本身就是实力"**:一个「我造了 40+ 个 AI 应用」的实时计数与作品墙

### ❌ 排除(公司/IMS 特有,保持私有,不进个人品牌)
ims-*、packet-core-wiki、ttcn_code、bcss-devportal、cnims-day0-toolkit、udm-ai-sharing、presales-chatbot、delivery-issue-platform、operation-deploy、nir-report-ai、benchmark-review、trace-reviewer、rogers-audit、fd2-ai-framework、int-testcase-configurator、smm-config、pm-xml-diff、storyline-viva-engage、Arch Reliability/redundancy-design、Learning IMS、IMS AI Engineering Matrix 等。
> 客户交付类课程(maritime-ai-course、ntt-course 等)如含客户信息也需脱敏或排除。

### ⚠️ 现站点必须修的硬伤(为什么要重做)
1. **假"实时资讯"**:`/api/news` 定义了 4 个新闻源却从不使用,返回一段**写死的 25 条**新闻只盖当天日期;前端还显示"数据源/每5分钟更新"。名不副实,且含虚构未来事件,伤信任 → **必须换成真自动化或删掉**。
2. **作品无截图**:`products.ts` 里 `image` 全是空字符串 —— 作品墙没有视觉,吸引力大打折扣。
3. **README 还是脚手架默认模板**;存在死代码(未用变量),ESLint 会报警。
4. **定位生硬、信息架构零散**(products/tools/legal/opc/skills 各自为政,没有品牌叙事线)。

## 三、信息架构(网站地图)

把零散板块重组成一条清晰叙事线:**认识珂 → 看珂造的东西 → 学珂的方法 → 找珂合作**。

```
首页 /                     品牌主张 + 40+作品计数 + 三支柱 + 精选作品 + 最新手记 + CTA
├─ 作品 /works             ★核心★ 可筛选作品墙(带真实截图/live链接/技术栈/"怎么造的")
│   分组: 原创应用 · Skills · 法律AI · 一人公司 · 精选Fork
├─ Skills /skills          技能蒸馏馆:女娲/仓颉旗舰叙事 + Skill 库 + 一键获取
├─ 法律AI /legal           法律人专属落地页:文书分析/诉讼Agent/专利/合规 + 咨询CTA
├─ 一人公司 /solo          一人公司AI装备:报税/合同/合规/业务 + "我如何用AI单干" + 服务
├─ 培训 /training          ★变现★ 通用/教培/物流课程 + 形式 + 案例 + 预约CTA
├─ 手记(博客) /blog        ★内容引擎★ MDX:造物记/方法论/法律AI/一人公司/资讯观察 + RSS
├─ AI情报 /news            真·自动生成的每日/每周AI摘要(见技术方案),或并入手记周报
├─ 关于 /about             真实故事:法律人→AI独立开发者;时间线/能力/联系
└─ 合作&订阅               线索捕获(培训/咨询/定制) + 邮件订阅 + 公众号二维码
```

设计取舍:`tools`(第三方工具库)**降级**为博客里的一篇"我常用的AI工具"或从导航移除——它不体现你的独特价值,别占一级入口。

## 四、内容策略(内容飞轮)

**每造一个东西 → 写一篇「造物记」→ 分发到多平台 → 引流回站 → 订阅/转化。**

- **造物记(Build Log)**:每个项目一篇"我为什么造、怎么造、踩了什么坑、你怎么用",是你最真实、最能立人设的内容,也天然是作品墙的详情页。
- **方法论**:女娲/仓颉的蒸馏方法、Claude Code / Skill / Agent 工程化经验 —— 你的差异化知识资产。
- **法律AI / 一人公司专栏**:垂直深度,吸引高价值受众。
- **AI情报**:用你自己的 Agent 能力做一个**透明自动化**的每日/每周摘要("由珂调教的AI自动生成"),既有用又是活广告。
- **分发中枢**:主站是大本营,内容一鱼多吃到 公众号 / B站 / 小红书 / 掘金 / 知乎 / X;主站聚合并回链。

## 五、设计与视觉系统

- **调性**:好玩但可信。抛弃现在的灰白公司脸,建立一个有记忆点的视觉:一个主强调色 + 大胆的中文标题排版 + 适度动效(framer-motion 已在)。
- **品牌元素**:字标"**珂以**"(珂字高亮/做成 monogram)+ slogan「珂以这样玩AI」;可考虑一个围绕"珂"的极简吉祥物/印章。
- **作品卡系统**:统一的卡片(截图 + 一句话 + 标签 + live/GitHub + 热度),整站一致。
- **深浅色双主题 + 中文优先排版**(好的 CJK 字体)+ 全面响应式。

## 六、技术架构

保留现有底座(Next.js 16 App Router + TS + Tailwind v4 + framer-motion),做以下升级:

1. **内容层 → MDX**:用已装的 next-mdx-remote + gray-matter,把手记做成 `content/*.mdx`(frontmatter 元数据),支持代码高亮、封面图、标签。
2. **作品数据 + 自动截图**:保留 TS 数据文件,但补齐截图 —— 用你自己的 openscreen / excalidraw-recorder 或一个 Playwright 脚本**自动批量截图**所有 live 应用(自己吃自己狗粮,也是内容素材)。
3. **真·资讯管线**(替换假 API):Vercel Cron / GitHub Action 每日跑 → 拉取真实 RSS(机器之心/量子位/36氪AI/Hacker News 等)→ 用 Claude API 去重/分类/摘要 → 存 JSON 或 KV → 前端读取。彻底删掉写死数组和死代码。
4. **GitHub 同步(可选)**:服务端定时拉取你的公开仓库列表/star 数,保持作品墙自动更新(注意用 token 避免限流)。
5. **SEO**:每页独立 metadata、`sitemap.xml`、`rss.xml`、每个作品/文章的 OG 图、结构化数据(Person/Article schema)、兼顾百度。
6. **分析**:Vercel Analytics 或自托管 Umami。
7. **域名**:主场用你已有的 **keaimentor.com**(与"珂爱/珂"呼应),把散落的子域应用统一从主站作品墙链接过去;或注册一个与品牌更贴的新域。
8. **国际化(可选)**:英文版扩大技术圈影响力。

## 十一、已锁定的决策(2026-07)与侧重调整
- **首要目标 = 涨粉 / 建立影响力**
- **落地 = 现站(Next.js 16)上重构**
- **主域 = keaimentor.com**
- **语言 = 中文优先(暂不做英文)**

### 因"影响力优先"带来的侧重调整
引擎从"转化落地页"切换到"**作品展示 + 内容飞轮 + 分发/订阅**":
1. **作品墙 + 造物记**是核心增长资产 —— 强人设("我造了40+个AI应用")+ 每个作品可分享(高质量 OG 图)。
2. **手记(MDX 博客)**尽早启动,3–5 篇造物记打底,一鱼多吃分发到公众号/B站/小红书/掘金/X。
3. **AI情报自动摘要**作为"每周回访钩子",也是活广告。
4. **关注/订阅入口**(公众号二维码 + 邮件订阅)前置到首页显眼位 —— 把流量沉淀为可反复触达的自有受众。
5. 法律AI / 一人公司 / 培训 三个纵向页**降为 Phase 2**(仍在,用于深度与背书,但不是当前转化重心)。

### 影响力优先的 Phase 1 冲刺顺序
① 首页(强人设+40+计数+精选作品+关注CTA)→ ② 作品墙(补真实截图+分享优化)→ ③ 手记 MDX + 3–5 篇造物记 → ④ AI情报真自动摘要 → ⑤ SEO/OG/RSS/分析 → ⑥ 订阅/公众号入口。
