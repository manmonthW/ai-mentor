每次做 Agent 相关的分享,总有人问我同一个问题:

「说了这么久 Agent、Skill,到底有什么好用的?为什么我到现在一个好用的 Agent 都没用过?」

这个问题特别真实。我自己也在公司内部喊了很久 Agent,一度也没有一个大家公认好用的。后来我想明白了,这里面有一个被绝大多数人忽略的关键。

## 最好用的,一定是定制的

先纠正一个错误预期:**不存在一个「装上就好用」的通用 Agent。**

无论是 Agent 还是 Skill,好用的前提只有一个——**它是基于你自己的需求定制的**。你不需要从零开始造,完全可以站在别人的经验和现成的 Agent 之上,但一定要加进去你自己的需求、你自己的工作流、你自己的口味。改到贴身为止,它才会变成你觉得「离不开」的东西。

这就像西装:成衣永远不如量体裁衣。区别在于,AI 时代「裁缝」这个角色,你自己就能当。

## 「最后一公里」理论

去年很多通用 Agent 产品火起来的时候,舆论也是两个极端:一派吹得神乎其神,一派说它啥也不是。

我试用之后的结论是:**这类工具解决的是「最后一公里」的问题,而最后一公里只对已经走完前面九十九公里的人有价值。**

那些把通用 Agent 用得特别好的人,无一例外,是前期对 AI 的掌握已经炉火纯青的人:他们熟练使用 Claude Code,自己写 Skill,自己搭工作流,想法能随手变成工具。对他们,Agent 是如虎添翼;对还没入门的人,Agent 只是一个更贵的聊天框。

还有一点容易被忽略:这类框架本身并不聪明,**它的聪明程度取决于背后调用的模型**。同一个 Agent 框架,接上顶级模型和接上凑合的模型,是两个完全不同的产品——前者能理解任务、正确调用工具、遇到问题自己想办法;后者连你的任务都理解不对。工具可以省,模型不能省。

## 那该怎么开始?

如果你现在一个好用的 Agent 都没有,我的建议不是去找「最好的 Agent 推荐清单」,而是:

1. **从你自己最烦的一件小事开始。** 每周要写的周报、要汇总的邮件、要整理的记录——挑一个;
2. **先用现成的,再改成自己的。** 找一个接近的 Skill 或工作流拿来就用,用的过程中把不顺手的地方一点点改掉;
3. **改完这一个,你就会了。** 定制第一个 Agent 的过程,会教会你定制之后所有 Agent 的方法。

我自己所有觉得好用的 Skill——发公众号的、做日报的、生成漫画的、写方案的——没有一个是「下载即用」的,全都是拿来之后按自己的流程改过好几轮的。

**Agent 不是买来的家电,是养出来的手感。** 这就是为什么你还没用上一个好用的——因为它还在等你动手。
