10 月 16 日,Anthropic 发布了 **Agent Skills**。我断断续续玩了两天,现在可以下结论了:爱不释手。

## Skills 是什么:给 Claude 的「岗位手册」

背景先讲清楚。所谓一个 Skill,本质上就是**一个文件夹**:里面放一份 `SKILL.md`,写清楚这件事该怎么做——步骤、标准、注意事项;需要的话,还可以附上脚本和参考资料。

巧妙的地方在于**按需加载**:Claude 平时只看到每个 Skill 的名字和一句话简介,几乎不占上下文;只有当任务真的匹配时,它才把整份手册翻开来读。Anthropic 自己就用这套机制做了处理 Word、Excel、PPT 的官方 Skills,而且它在 Claude 应用、Claude Code 和 API 里通用。

发布当天,Simon Willison 就写了篇文章说:Skills 可能是比 MCP 更大的事。我试完的感受是——他没夸张。

## 为什么我一上手就停不下来

因为它解决的正是我最头疼的问题:**好不容易调教出来的做事方法,留不住。**

以前每次让 AI 干活,那些「按我们行业的规矩来」「这一步要注意什么」的经验,都得在对话里重新交代一遍。对话一关,一切归零。Prompt 越写越长,但它终究是一次性的。

Skill 把这件事变成了**沉淀**:方法写进文件,文件放进文件夹,从此每次干活它都自己带上。我试着把做 PPT 的一套流程写成 Skill,第二次调用时那种「它记得」的感觉,就是我停不下来的原因。

## 一个更大的念头

玩到第二天,一个念头冒出来,越想越兴奋:

> 既然「怎么做一件事」可以写成 Skill——那**一个人的思维方式**,能不能?**一本经典名著里的方法论**,能不能?

如果能,那 Skill 就不只是给 AI 的岗位手册,而是一种**蒸馏装置**:把人脑子里、书页里那些只可意会的东西,蒸馏成可执行、可复用、可传承的能力。

这个方向我要认真挖下去。挖出什么来,会放在[手记](/blog)里。
